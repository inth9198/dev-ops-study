# Memory Bank

## 중요 작업 원칙

### 문제 해결 접근 방식

**절대 지켜야 할 규칙:**

1. 문제가 발생했을 때 → **왜 안 되는지 이유를 먼저 설명**
2. 다른 해결 방법을 **제시만** 할 것 (원리와 함께 설명)
3. **말없이 다른 방법을 시도하지 말 것** - 사용자에게 설명 후 제시
4. 실제로 실행하지 않고 방법만 제시

**예시:**

- ❌ 잘못된 방식: 바로 다른 명령어를 실행해봄
- ✅ 올바른 방식: "이 방법은 XX 이유로 안 됩니다. 대신 YY 방법을 사용할 수 있는데, 이는 ZZ 원리로 작동합니다."

### 2024 - 프로세스 생성 문서 작성

- **파일**: `2장/프로세스 생성.md`
- **작업 내용**: 프로세스를 생성하는 목적 설명 추가
  - 멀티태스킹, 작업 분리, 병렬 처리, 서비스 제공, 프로세스 계층 구조, 프로그램 실행 등 6가지 주요 목적 설명

- **작업 내용**: fork() 함수 상세 설명 추가
  - fork()의 개념과 원리
  - 동작 방식 (프로세스 복제, 반환값, 실행 흐름)
  - 함수 시그니처 및 기본 사용 예시
  - fork()의 특징 (메모리 복사, 파일 디스크립터 공유, 독립적인 실행, 프로세스 ID)
  - 실제 사용 시나리오 (멀티프로세스 서버, 병렬 작업 처리, 새 프로그램 실행)
  - 주의사항 (무한 루프, 자식 프로세스 정리, 메모리 사용량, 동기화)

- **작업 내용**: 프로세스 관련 주요 개념들 추가
  - execve() 함수: 개념, 원리, 동작 방식, 함수 시그니처, 사용 예시, exec 계열 함수들, fork()와의 조합
  - ASLR (Address Space Layout Randomization): 개념, 원리, 보안 효과, 구현 방식, 확인 방법
  - 프로세스의 부모-자식 관계: 개념, 관계의 특징, PID/PPID, 리소스 상속, 프로세스 계층 구조, 확인 방법
  - 프로세스 상태: Running, Sleeping, Uninterruptible Sleep, Stopped, Zombie 상태 설명 및 상태 전환, 확인 방법
  - 프로세스 종료: 정상 종료, 시그널에 의한 종료, 에러로 인한 종료, exit() 함수, 종료 상태 확인
  - 좀비 프로세스와 고아 프로세스: 개념, 원인, 문제점, 해결 방법, 확인 방법
  - 시그널: 개념, 원리, 주요 시그널 목록, 시그널 전송, 시그널 핸들러 등록, 시그널 블로킹
  - 셸 작업 관리 구현: 포그라운드/백그라운드 작업 개념, 작업 관리 명령어, 구현 원리
  - 세션: 개념, 세션 리더, 제어 터미널, 세션 생성, 확인 방법
  - 프로세스 그룹: 개념, 프로세스 그룹 ID, 그룹 리더, 시그널 전송, 프로세스 그룹 생성 및 확인
  - 데몬: 개념, 특징, 데몬 생성 과정 (8단계), 확인 방법, 주요 데몬 예시
