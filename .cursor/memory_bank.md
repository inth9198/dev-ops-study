# Memory Bank

## 중요 작업 원칙

### 문제 해결 접근 방식

**절대 지켜야 할 규칙:**

1. 문제가 발생했을 때 → **왜 안 되는지 이유를 먼저 설명**
2. 다른 해결 방법을 **제시만** 할 것 (원리와 함께 설명)
3. **말없이 다른 방법을 시도하지 말 것** - 사용자에게 설명 후 제시
4. 실제로 실행하지 않고 방법만 제시

**예시:**

- ❌ 잘못된 방식: 바로 다른 명령어를 실행해봄
- ✅ 올바른 방식: "이 방법은 XX 이유로 안 됩니다. 대신 YY 방법을 사용할 수 있는데, 이는 ZZ 원리로 작동합니다."

## 작업 이력

### 2024 - 리눅스 개요 문서 업데이트

- **파일**: `리눅스 개요.md`
- **작업 내용**: 커널(Kernel)에 대한 설명 추가

  - 커널의 정의 및 역할 설명
  - 주요 기능 목록 추가 (하드웨어 관리, 프로세스 관리, 메모리 관리, 파일 시스템 관리, 시스템 호출, 보안)
  - 커널의 위치 및 사용자 공간/커널 공간 분리 개념 설명

- **작업 내용**: 시스템 콜(System Call)에 대한 설명 추가

  - 시스템 콜의 정의 및 역할 설명
  - 주요 특징 (권한 전환, 보안, 추상화)
  - 주요 시스템 콜 종류 (파일 조작, 프로세스 관리, 메모리 관리, 네트워크, 시스템 정보)
  - 시스템 콜 동작 과정 (6단계)
  - C 언어 예시 코드 추가

- **작업 내용**: CPU 모드(커널 모드와 사용자 모드)에 대한 설명 추가

  - 사용자 모드의 정의, 권한 수준, 제한 사항 설명
  - 커널 모드의 정의, 권한 수준, 가능한 작업 설명
  - 모드 전환 시점 및 과정 설명
  - 두 가지 모드가 필요한 이유 (보안, 안정성, 자원 보호, 멀티태스킹)
  - 모드 전환 과정 다이어그램 및 예시 시나리오 추가

- **작업 내용**: 시스템 콜 추적 도구에 대한 설명 추가

  - Linux의 strace 도구 설명
  - macOS의 dtruss 도구 설명 및 제한사항
  - "DTrace cannot instrument translated processes" 오류 분석
  - 오류 발생 이유 (SIP, 보안 메커니즘, 바이너리 특성)
  - Linux와 macOS의 시스템 콜 추적 도구 비교표 추가

- **작업 내용**: 시스템 도구 개념 설명 섹션 추가 (비교 중심에서 개념 설명 중심으로 변경)

  - 시스템 성능 모니터링 도구 개념 설명 (sar, vm_stat, iostat, top/htop)
    - 각 도구의 개념, 원리, 주요 기능, 사용 예시
  - 라이브러리 의존성 확인 도구 개념 설명 (ldd, otool -L)
    - 각 도구의 개념, 원리, 동적/정적 링킹 개념
  - 패키지 관리 도구 개념 설명 (dpkg/apt, brew)
    - 각 도구의 개념, 원리, 주요 명령어
    - 시스템 라이브러리 위치 정보
  - **작성 원칙**: Linux vs macOS 비교보다는 각 명령어의 개념적 설명과 내용에 초점

- **작업 내용**: 시스템 콜 래퍼 함수 설명 추가

  - 시스템 콜 래퍼 함수의 개념과 원리

  - 래퍼 함수의 동작 과정 (상세한 단계별 설명)
  - 실제 예시 코드와 내부 동작 설명

  - 정적 라이브러리의 장단점
  - 공유 라이브러리(동적 라이브러리)의 개념, 원리, 파일 형식
  - 공유 라이브러리의 장단점
  - 동적 로딩 과정 상세 설명

  - 실제 사용 예시 코드 및 명령어
